\## Задача 1



\### Текст задачи



Разработать класс с тремя целочисленными полями. Создать конструктор копирования.

Разработать метод, вычисляющий произведение полей. Перегрузить метод ToString() для

формирования строки из полей класса. Реализовать дочерний класс (его содержание предложить

самостоятельно и описать в решении: какой содержательный смысл имеют поля; реализовать

конструкторы; предложить и реализовать 2-3 метода). Протестировать все конструкторы и

другие методы базового и дочернего классов.



\### Алгоритм решения



1)Попадаем в функцию Main.

2\) Три раза вызываем функцию ReadPositiveInteger для ввода длины,ширины и высоты. В функции считываем строку, проверяем является ли ввод целым числом больше нуля. Если д, то возвращаем число, если нет , то выводим ошибку и повторяем ввод.

3\) Создаем объект baseBox через конструктор с параметрами и выводим на экран.

4\) Вычисляем и выводим объем методом CalculateVolume

5\) Создаем дубликат copiedbasebox через конструктор копирования.

6\) Четыре раза вызываем ReadPositiveInteger для ввода габаритов и веса, а материал считываем напрямую через Console.Readline.

7\) Создаем объект heavyBox, дочерний конструктор передает габариты в базовый класс через :base, а вес и материал созраняет сам, выводим. Выводим объем через унаследованный метод CalculateVolume. Вычисляем и выводим плотность методом CalculateDensity. Проверяем вес методом IsTooHeavyForHuman. Если вес > 20000 г, выводим предупреждение, иначе строку о возможности ручного подъема. Создаем и выводим дубликат copiedHeavyBox через дочерний конструктор копирования.

8\) Завершаем программу

\# Задание 2, 3



\## Задача 2, 3



\### Текст задачи



В задании 2 необходимо реализовать определение класса (поля, свойства, конструкторы, перегрузка

метода ToString() для вывода полей, заданный метод согласно варианту). Протестировать все

методы, включая конструкторы (при тестировании методов классов в заданиях не забывайте

проверять вводимые пользователем данные на корректность). (Time, byte hours

byte minutes, Вычитание переменной типа Time (учесть, что возможен переход в предыдущие сутки). Результат должен быть типа Time.)



В задании 3 добавить к реализованному в задании 2 классу указанные в варианте перегруженные

операции. Протестировать все методы.

(Унарные операции:

++ добавление минуты к объекту типа Time.

\-- вычитание минуты из объекта типа Time

Операции приведения типа:

int (неявная) – результатом является количество минут (время

переводится в минуты);

bool (явная) – результатом является true, если часы и минуты не равны

нулю, и falseв противном случае.

Бинарные операции:

< Timet1, Timet2 — время переводится в минуты, результатом является

true, если количество минут в левом операнде меньше, чем количество

минут в правом операнде, и false–в противном случае.

> Timet1, Timet2 — время переводится в минуты, результатом является

true, если количество минут в левом операнде больше, чем количество

минут в правом операнде, и false–в противном случае)



\### Алгоритм решения



1\) Попадаем в функцию main

2\) С помощью метода ReadByteInput запрашиваем данные для двух объектов времени t1 и t2. В методе с помощью byte.TryParse проверяем, является ли ввод числом и входит ли оно в заданные границы (0–23 для часов, 0–59 для минут). Если да, то возвращаем число, если нет, то выводим ошибку и повторяем опрос.

3\) Создаем объекты t1 и t2 через конструктор с параметрами. Защита в свойствах Hours и Minutes дополнительно страхует логику с помощью операции %. Выводим объекты на экран. Автоматически срабатывает метод ToString, форматирующий время в вид ЧЧ:ММ. Создаем дубликат первого времени tCopied через конструктор копирования и выводим его. Вызываем метод вычитания t1.Subtract(t2). Внутри метода переводим оба времени в абсолютные минуты. Если разница уходит в минус, прибавляем сутки в минутах (24 \* 60). Переводим результат обратно в часы/минуты и выводим на экран. 

4\) Проверяем унарный инкремент (t1++) и декремент (t1--). Операторы переводят время на 1 минуту вперед/назад с контролем перехода через полночь и возвращают измененный объект. Проверяем неявное приведение типа к int (int totalMinutes = t1;). Оператор implicit автоматически рассчитывает и возвращает общее количество минут с начала суток. Проверяем явное приведение типа к bool ((bool)t1). Оператор explicit возвращает true, если время не равно 00:00, и false в противном случае. Проверяем бинарные операторы сравнения (t1 > t2 и t1 < t2). Они переводят оба объекта времени в минуты, сравнивают полученные числа и возвращают результат логического сравнения.

5\) Завершаем программу.

Скриншоты результата работы программы
